# Agent Spec

## Overview
The `customer-support-agent` is a graph-based workflow agent built using ADK 2.0. It acts as a customer support representative for a shipping company. The workflow classifies user queries to determine if they are related to shipping (rates, tracking, delivery, returns) or unrelated. Shipping-related queries are routed to a dedicated `LlmAgent` loaded with mock shipping FAQ policies. Unrelated queries are routed to a dedicated node that politely declines to answer.

## Example Use Cases
1. **Shipping Query - Rates:**
   - **Input:** "How much does it cost to ship a 5lb package to New York?"
   - **Expected Behavior:** Routed to Classifier -> Routed to Shipping FAQ Agent.
   - **Expected Output:** Shipping FAQ Agent responds with the shipping rates (e.g., $5 flat rate for standard shipping).

2. **Shipping Query - Tracking:**
   - **Input:** "Where is my package? My tracking number is TRK123456."
   - **Expected Behavior:** Routed to Classifier -> Routed to Shipping FAQ Agent.
   - **Expected Output:** Shipping FAQ Agent provides tracking instructions and details.

3. **Unrelated Query:**
   - **Input:** "What is the capital of France?"
   - **Expected Behavior:** Routed to Classifier -> Routed to Decline Node.
   - **Expected Output:** Polite response stating the agent can only assist with shipping queries.

## Tools Required
None. The shipping FAQ agent uses static mock policies built into its system instructions.

## Constraints & Safety Rules
- The agent must ONLY answer shipping-related queries (rates, tracking, delivery, returns).
- For unrelated queries, the agent must politely decline to answer without providing information on the unrelated topic.
- Use `gemini-3.5-flash` for LLM nodes.
- No deployment target needed (scaffolded with `--prototype`).

## Success Criteria
- 100% correct classification of shipping vs. unrelated queries.
- Polite refusal for unrelated queries.
- Helpful, accurate answers (based on mock policies) for shipping-related queries.

## Reference Samples
None. Standard graph workflow implementation.
